Architecture

How it works

A bridge extension inside VSCode exposes local agent access backed by vscode.lm. Editor tools, Copilot-visible MCP, and project skills flow into the same terminal runtime.

01 — Your machine
Your Terminal
Run omc in any directory. The CLI connects to the bridge over localhost. Your files never leave your machine.
localhost only
→
02 — Adapter layer
Bridge Extension
A lightweight VSCode extension that translates OpenAI-style requests into vscode.lm API calls. Auto-starts with VSCode.
vscode.lm API
→
03 — Model provider
GitHub Copilot
Your existing Copilot subscription. GPT-4o, Gemini, o1, and other models included in your plan. No separate API key, no extra billing.
your subscription
Code stays on your machine. Only the prompt reaches Copilot — same as typing in the editor.
Execution modes

Four modes

Switch with /mode or natural language via /auto_set.

?
ask
Direct Q&A. No tools invoked. Fastest response for questions, explanations, and quick lookups.
#
plan
Read-only analysis. Proposes a structured plan without making any file changes. Review before committing.
>
agent
Full autonomous mode. Uses all tools — read, edit, run commands. Signals DONE when the task is complete.
>>
ultrawork
Maximum autonomy. Enforces explore → plan → execute → verify. Oracle reviews the output; the agent fixes issues up to 3 rounds.
Specialized agents

Nine agents

Each agent has a distinct execution pattern. Switch with /agent or Tab, then layer skills and MCP on top.

basic
Primary · Minimal
Closest to plain Copilot Chat behavior. Best when you want low-overhead help inside the same terminal harness.
sisyphus
Primary · Default
Primary orchestrator. Plans, executes, verifies, and now treats visible Copilot editor capabilities like search, docs, and GitHub tools as built-in leverage.
hephaestus
Primary · Deep worker
Explores the entire codebase before touching a single file. Best for large changes that span many files or browser-heavy UI work.
prometheus
Primary · Planner
Interview-driven. Asks the right questions, then produces a structured step-by-step execution plan.
atlas
Specialist · Orchestrator
Builds an explicit checklist, executes it fully, and self-verifies at each step. Heavy but thorough.
oracle
Specialist · Reviewer
Read-only architecture advisor. Used as the ultrawork verifier. Critical, skeptical, precise.
explore
Specialist · Search
Parallel codebase search. Runs 3+ simultaneous searches and returns structured, actionable results.
metis
Specialist · Scope guard
Pre-planning reviewer. Catches AI over-engineering before a single line is written.
momus
Specialist · Plan verifier
Checks that a plan is actually executable. Approval-biased — only rejects genuine blockers.
librarian
Specialist · Docs
Documentation researcher. Finds usage patterns and explains external libraries from source.
Get started

One command.
Everything set up.

Install the CLI, open VS Code or Cursor, then work mostly inside the TUI with agents, skills, MCP, and Copilot editor tools.

VSCode 1.90+ with GitHub Copilot
Node.js 18+
macOS, Linux, or Windows (WSL)
Cursor also supported
1
Install via npm
npm install -g github:JJongyn/oh-my-copilot
Builds the CLI and installs the bridge extension into VSCode or Cursor automatically.
2
Open VSCode
The bridge extension starts automatically. Verify via Cmd+Shift+P → "Oh My Copilot: Show Bridge Status" — should show a green port number.
3
Run the CLI
omc
Then start with /init, /skills, /agent basic, or /agent sisyphus. If VSCode is not in PATH, the installer prints the manual bridge install command.
